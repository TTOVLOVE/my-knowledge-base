# 🏛️ 远程会话与安全审计平台：架构设计

本系统旨在解决企业内网运维过程中的权限管控混乱、高危操作无法溯源等痛点。通过构建统一的远程访问网关，实现对 SSH、RDP 等协议的细粒度会话管理与全量安全审计。

## 📊 系统核心架构图

系统整体采用**前后端分离 + 核心代理网关**的架构模式，自上而下划分为四个逻辑层：

1. **Web 表现层 (Frontend)**
   * 提供管理员控制台与用户视图。
   * 负责资产展示、策略配置、以及在线 Web 终端（Web Terminal）的直连访问。
2. **业务控制层 (Control Plane)**
   * 处理用户认证（鉴权与 SSO 接入）。
   * 管理资产授权策略（RBAC 模型）。
   * 调度审计任务与日志下发。
3. **协议代理网关 (Data Plane / Proxy Gateway)**
   * **这是系统的核心引擎。** 负责拦截、解析并转发客户端与目标服务器之间的底层协议流量（如 SSH, RDP, Telnet）。
   * 实时阻断触发安全规则的高危命令。
4. **数据与审计存储层 (Storage)**
   * **关系型数据：** 存储用户、资产、授权关系。
   * **审计录像：** 采用高效的结构化格式存储终端录像（如 asciinema 格式），以便于极速回放和全文命令检索。

## 🛡️ 核心业务流转：一次安全的 SSH 登录

为了实现无缝且安全的审计，系统设计了如下会话流转机制：

1. 用户登录 Web 控制台，获取被授权的资产列表。
2. 用户点击“连接”，Web 端向控制面申请一个临时会话 Token。
3. 控制面将 Token 与目标资产凭证下发给 **协议代理网关**。
4. 代理网关利用 WebSocket 与浏览器建立连接，同时在后端使用真实凭证发起向目标服务器的 SSH 连接。
5. **审计开启：** 代理网关在中间实时双向抓取 I/O 数据流，异步写入审计存储模块，实现“所见即所录”。
