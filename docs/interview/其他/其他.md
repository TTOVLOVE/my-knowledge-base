# 网络安全工程师

## 一、 蓝队与应急响应：攻击溯源 (Traceability)

攻击溯源就像网络世界的侦探，通过收集和分析线索，找出攻击者并还原攻击过程，最终保护网络安全。

- **核心目标**：识别攻击者、了解动机和路径、评估损失、加固防御、法律诉讼和情报共享。核心是找到攻击来源，阻止进一步危害。
- **基本流程**：事件响应与取证 (数据收集) ➔ 日志分析 ➔ 网络流量分析 ➔ 恶意软件分析 ➔ 威胁情报分析 ➔ **攻击者画像 (溯源的核心)** ➔ 报告与共享。（这是一个迭代的过程）
- **关键技术**：日志管理/分析、网络流量分析、恶意软件分析、威胁情报、行为分析、取证、沙箱、机器学习/AI。
- **常用工具**：
  - 日志分析：Splunk, ELK
  - 流量分析：Wireshark, Zeek
  - 恶意软件分析：Cuckoo, IDA Pro
  - 威胁情报：Recorded Future, VirusTotal
  - 取证工具：EnCase
- **面临挑战**：数据量大且分散、攻击者伪装、恶意软件复杂、隐私保护、跨境调查、法律合规、专业技能与资源缺乏。

## 二、 Web渗透与漏洞原理

## 1. Web 渗透基本流程 & 测试思路

- **常规流程**：信息收集（主机探活，端口扫描，系统识别） ➔ 漏洞扫描/挖掘 ➔ 漏洞利用 ➔ 提权/维权（注意判断是否为站库分离）。
- **登录框测试思路**：
  - 弱口令爆破（Hydra、Burp）
  - 万能密码/SQL注入（如 `admin'--`）
  - 前端 JS 调试查找逻辑绕过、隐藏接口
  - 注册/忘记密码功能权限绕过
  - Cookie 伪造 / Token 猜测 / 暴力登录限速绕过
- **针对特定链接的测试（如 `demo.do?DATA=dWlkPTE2==`）**：
  - Base64 解码分析：解码后为 `uid=16`。
  - 尝试越权/逻辑漏洞：修改参数值为 `uid=17`，再重新 Base64 编码发包。
  - 尝试注入：构造 SQLi（如 `uid=1' or 1=1--`）或 XSS（如 `<script>alert(1)</script>`），编码后测试。
  - 检查潜在的二次反序列化或加密字段逻辑。

## 2. 核心漏洞解析

- **XSS (跨站脚本攻击)**
  - **效果**：窃取 Cookie/Token、劫持会话、页面伪造钓鱼、配合 CSRF 攻击、控制浏览器执行恶意 JS。
  - **防御**：输入校验 + 输出转义（HTML/JS/CSS）、配置 CSP（内容安全策略）、Cookie 设置 `HttpOnly` 和 `Secure`、使用主流框架自带的防 XSS 机制。
- **SSRF (服务器端请求伪造)**
  - **利用思路**：探测内网端口/服务（127.0.0.1、Redis 等）；打内网接口（K8s、Consul）；访问云服务元数据（IMDS，如 `http://169.254.169.254/`）；利用 Gopher/Dict 协议攻击内网 Redis；结合 `file://` 协议读取本地文件（如 `/etc/passwd`）。
- **OOB (带外注入)**
  - **原理**：利用数据库漏洞，通过让数据库服务器向**攻击者控制的服务器**发起网络请求（如 DNS 解析、HTTP 请求）来获取敏感信息的技术。它不直接从数据库的 HTTP 响应中获取数据，适合用于盲注场景。
- **反单引号 ``` 的作用**
  - **PHP**：表示执行系统命令，等同于 `shell_exec()`。例如：`echo `ls`;`
  - **SQL (MySQL)**：用于转义字段名、表名，防止与系统关键字冲突。例如：`SELECT `select` FROM `user`;`

## 三、 组件、框架与底层原理

## 1. Java 反序列化与反射机制

- **Java 反射的作用**：允许程序在运行时动态加载类、调用方法、访问属性（如 `Class.forName("java.lang.String");`）。在安全中，它是构造反序列化漏洞利用链（Gadget）的核心机制。
- **反序列化原理**：对象通过 `ObjectOutputStream` 序列化成字节流，再通过 `ObjectInputStream` 反序列化恢复。若类实现了 `Serializable` 且其魔术方法（如 `readObject()`、`readResolve()`、`finalize()`）中包含恶意逻辑，反序列化时会自动触发，导致 RCE。

## 2. Apache Shiro 550 反序列化漏洞

- **概念**：Shiro 是一个强大的 Java 安全框架，提供认证、授权和会话管理。
- **漏洞原理**：Shiro 提供了 `RememberMe` 功能。服务端接收 Cookie 后，处理流程为：**Base64 解码 ➔ AES 解密 ➔ 反序列化**。
- **利用方式**：在 `<=1.2.4` 版本中，AES 加密的 Key 是**硬编码**的。攻击者提取该 Key，将恶意命令（如 CC 链）序列化 ➔ AES 加密 ➔ Base64 编码，将其作为 `rememberMe` 的 Cookie 发送，服务端解密并反序列化后触发 RCE。常用工具：ysoserial、ShiroExploit。

## 3. Redis 未授权访问漏洞

- **原理**：Redis 配置不当，未开启密码认证或绑定在公网（0.0.0.0），任何人皆可直接连接执行命令。
- **利用方式**：
  - 写入 SSH 公钥到 `/root/.ssh/authorized_keys`，实现免密登录。
  - 写入计划任务（Cron）反弹 Shell。
  - 若知晓绝对路径，可写入 WebShell。

## 4. Swagger API 文档泄露测试

- 扫到 `swagger.json` 后：使用 Postman 或 Swagger UI 导入接口进行可视化测试；利用 Knife 等插件进行一键扫描；寻找未授权访问的敏感接口（文件上传、命令执行）；检查版本已知漏洞。

## 四、 内网渗透与域控安全

## 1. 内网主机探活 (获取边界权限后)

- **命令/工具**：
  - `arp -a`：查看同网段设备 ARP 缓存。
  - `ping` 或循环 Ping 脚本。
  - `nmap -sn 192.168.x.0/24`：仅发送 ICMP/ARP 包探测存活，不扫描端口。
  - `nbtscan`：扫描 NetBIOS 名称（Windows 环境）。
  - `smbclient -L`：探测 Windows 共享。
  - `masscan`：大段 IP 快速端口探测。

## 2. 域控票据：白银票据 vs 黄金票据

| **对比项**   | **黄金票据 (Golden Ticket)**         | **白银票据 (Silver Ticket)**                        |
| ------------ | ------------------------------------ | --------------------------------------------------- |
| **本质**     | 伪造 **TGT** (身份票据)              | 伪造 **SGT/TGS** (服务票据)                         |
| **利用前提** | 需获取域控 `krbtgt` 账号的 NTLM Hash | 需获取目标**机器/服务**账户的 NTLM Hash             |
| **权限范围** | 针对域内**所有机器的所有服务**       | 仅针对**特定机器的特定服务** (如 CIFS, HTTP)        |
| **日志痕迹** | 会在域控留下日志交互                 | 仅在目标系统留下日志，**不与 KDC 交互**，隐蔽性更强 |

- **关键日志 ID**：防御方应重点关注事件 ID `4624` (账户登录)、`4634` (账户注销)、`4672` (管理员登录)，异常时域字段可能为空。

## 3. 系统计划任务 (Cron) 目录

- **Linux/Unix**：
  - 用户任务：`/var/spool/cron/crontabs/<用户名>`
  - 系统任务：`/etc/crontab`、`/etc/cron.d/` 以及 `cron.hourly/daily/weekly/monthly` 目录。
- **Windows**：
  - 配置信息存储在**注册表**中（非文件系统）：`HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Schedule\TaskCache\Tree`。可使用“任务计划程序”管理。

## 五、 工具与基础技能

## 1. sqlmap 核心参数与原理

- `--technique`：指定注入技术。`B` (布尔盲注)、`T` (时间盲注)、`E` (报错)、`U` (联合)、`S` (堆叠)。
- `--dbms`：指定数据库类型（如 MySQL, MSSQL），加快速度防误报。
- `--os-shell` 原理：
  - **MySQL**：利用 `select ... into outfile` 向拥有写权限的 Web 目录写入 WebShell。
  - **MSSQL**：利用内置存储过程 `xp_cmdshell` 执行系统命令并反弹 Shell。
